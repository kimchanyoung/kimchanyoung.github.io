DBC Cheat-Sheet

In this cheat-sheet, we will cover several random topics like JavaScript Prototypes, object creation, and looping, as well as some basic CSS positioning.

JavaScript Objects

Objects in JavaScript is probably a good place to start.

In JavaScript, an object is a fundamental data type that holds an unordered list of properties as a series of name-value pairs. An object's properties are either primitive data types (number, string, boolean, undefined, or null) or methods. A unique object is usually created in one of 2 different ways -- by using the object literal notation, or by using the object constructor notation.

The object literal notation looks like this: 


            var pikachu = {
              name: Pikachu;
              id: 25,
              type: 'electric',
              function thundershock() {
                console.log("Pikachu has used Thundershock!");
              }
            };
It uses the var call to create one instance of a pikachu, which has the listed attributes. Alternatively, the object constructor method can be used as follows: 

            var pikachu = new Object() {
              name: Pikachu;
              id: 25,
              type: 'electric',
              function thundershock() {
                console.log("Pikachu has used Thundershock!");
              }
             }
As you can see, in the first line, var pikachu is instantiated as an instance of an Object. For reasons that we won't go into now, the object literal method is usually preferred.

When creating an object that will have different 'instances' of itself, the constructor pattern can be used. This makes one generic object that can be edited to make many different types of this object. 


            function Pokemon(theName, id_num, element_type) {
              this.name = theName;
              this.id = id_num;
              this.type = element_type;
              this.move = function() {
                console.log( this.name + " has used " + this.move + "!");
              };
            }

            var pikachu = new Pokemon('Pikachu', 25, 'electric');
This code creates a Pokemon object that can be used to create new instances of Pokemon, like a Pikachu.

Prototyping

In JavaScript, objects "inherit" certain properties and methods from one another. You can think of this as -- all JS objects have a dad, until its "Adam" -- "Object," which has a "null" parent. We saw in the previous section 3 different ways to create objects in JS. The first two, for unique objects, and the third, for a generic one to serve as a type. The first two methods create objects whose parent is the Object.prototype. They are direct descendants of the Object object. But an object created by the constructor pattern (like the third pikachu) is a direct descendant of the object from which it was constructed (Pokemon). Prototypes are like chains of inheritance in which each successor inherits the properties and methods of their predecessors.

Not only does this give all objects properties and methods from the ultimate prototype "Object," like hasOwnProperty(), isPrototypeOf(), and toString(), it allows for much control in creation of objects. For instance, if we wanted to make a Charizard object following the example in the constructor pattern example, we could write: 


          Charizard.prototype = new pokemon();

This feature in JavaScript necessitates a special piece of logic when cycling through an object's attributes. JavaScript allows for looping through an object's properties by calling: for (var key in obj). This code instantiates a variable "key" that will cycle through the attributes of an object "obj." But because of prototyping, you want to check to make sure that the key isn't the property of one of the object's prototypes! Therefore, After the for loop, before anything is actually done, all code has to be wrapped by the code if(obj.hasOwnProperty(key)). This bit of logic lets the code run only if the property is native to that object and not to one of its prototypes.
