RUBY CLASSES

A class is a thing. Just any 'thing.' Like a banana. Or a book. Or a person. These 'things' have features and attributes. Like color, or weight. And they do different things, like lay motionless, open, or walk.

Sorry, that's a bad explanation. Classes in Ruby are objects that can be given instance variables (think attributes) and instance methods (action verbs). New instances of classes can be made with a variable set of instance variables and instance methods to represent different things.

For example, let's say that we want to make a "banana" class. The code may look as follows. 


        Class Banana
          def initialize(color, length)
            @color = color
            @length = length
          end

          def peel
            @color = yellow
          end

          def eat
            (@length = @length - 1) unless (@length == 0)
          end
        end

In the preceding code, we have the definition of a "Banana" class. It takes in 2 variables for color and length, and receives that information as its attributes. It also has 2 instance methods that can be performed on an object of this class. You can peel the banana, which turns it yellow (bananas are yellow on the inside), and you can eat the banana, which decreases its length. You might want to create a thing to check whether the banana is peeled before you eat it (perhaps an instance variable that contains a boolean value peeled?) but let's just assume that we're barbarians who enjoy the bitter taste of peel and rubbery film on our teeth.

One could create a new banana named Frank by the following: frank = Banana.new("green", 7) This creates a new Banana named frank whose color is green and length is 7 (units, you barbarian). You can make it do things by calling frank.peel or frank.eat.
